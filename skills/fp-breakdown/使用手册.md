# fp-breakdown 使用手册

> 一个 Claude Code Skill，根据系统功能描述自动拆分软件功能点（IFPUG 方法），输出标准表格，并同时生成 `.md` 和 `.xlsx` 两份文件。投标、需求分析、工时估算场景必备。

---

## 一、这是什么

**fp-breakdown** 是一个 Claude Code 技能（Skill），它能：

- 把一段系统功能描述自动拆成符合 **IFPUG 功能点法**的功能点清单
- 按 **ILF / EIF / EI / EO / EQ** 五种标准类型分类，每类有标准功能点数
- 输出包含"功能需求 / 序号 / 类型 / 功能点数 / 功能点名称"的五列表格
- 同时生成 `功能点清单.md` 和 `功能点清单.xlsx` 两份文件到当前工作目录
- 末尾自动统计：合计 N 个功能点，共 M 功能点数

**典型用途**：

- 投标技术文件中的功能点估算
- 软件项目的工时与成本评估
- 需求分析阶段的需求拆解

## 二、安装前的准备

### 1. 必须项

- **Claude Code CLI** 已安装（VSCode 扩展支持有限，下面会说）
- 命令行能跑 `claude --version` 输出版本号

如果没装 Claude Code：

```powershell
npm install -g @anthropic-ai/claude-code
```

### 2. 可选项

- **Python + openpyxl**：用于生成 `.xlsx` 文件。若没有，skill 会跳过 xlsx 只输出 md，并提示你手动安装
  ```powershell
  pip install openpyxl
  ```
- **代理**（中国大陆用户）：从 GitHub 拉取 marketplace 时可能需要

## 三、安装方法

提供 **两种方式**，按你的环境选一种。

### 方式 A：通过 Plugin Marketplace 安装（推荐）

**前提**：你的 Claude Code 环境支持 `/plugin` 命令（独立 CLI 一定支持，部分 IDE 扩展不支持）。

1. **打开 PowerShell**（或终端），启动 Claude Code：
   ```powershell
   claude
   ```

2. **添加 marketplace**：
   ```
   /plugin marketplace add becatjd/claude-skills
   ```
   显示 `Successfully added marketplace: claude-skills` 即成功。

3. **安装 plugin**：
   ```
   /plugin install fp-breakdown@claude-skills
   ```
   在弹出的菜单中选择 **`Install for you (user scope)`**（全局可用），按回车。

4. **验证**：
   ```
   /plugin
   ```
   切到 **Installed** 标签页，应看到 `fp-breakdown · claude-skills · √ enabled`。

完成。重启 Claude Code 后即可在任何项目使用。

---

### 方式 B：手动复制安装（兜底方案）

适用于：你的 Claude Code 环境**没有** `/plugin` 命令（如某些 IDE 扩展），或不想用 plugin 系统。

1. **下载 SKILL.md**：

   - 从 GitHub 仓库 https://github.com/becatjd/claude-skills 下载 `skills/fp-breakdown/SKILL.md`
   - 或让仓库维护者直接把文件发给你

2. **放到全局 skills 目录**：

   - **Windows**：`C:\Users\<你的用户名>\.claude\skills\fp-breakdown\SKILL.md`
   - **macOS / Linux**：`~/.claude/skills/fp-breakdown/SKILL.md`

   目录不存在的话自己建一个 `fp-breakdown` 文件夹。

3. **重启 Claude Code**，skill 即被自动识别。

## 四、怎么用

### 触发方式：自然语言

安装后**不需要**任何特殊命令，直接在对话里描述需求即可。Claude 会自动识别并调用 fp-breakdown。

### 推荐 prompt 模板

```
帮我把这个 [系统名] 拆成 [数量] 个功能点：[功能描述]
```

### 实际示例

**示例 1：学生选课系统**

```
帮我把这个学生选课系统拆成 30 个功能点：
学生可注册登录、选课、退课、查成绩；
教师可发布课程、录入成绩、查看选课情况；
管理员管理用户、课程和学期；
系统对接学校统一身份认证平台。
```

**示例 2：投标场景**

```
我在做一个图书馆管理系统的投标，需要 50 个功能点的清单做工时估算。
系统包括：图书采编、流通借阅、读者管理、统计报表、与图书馆联盟数据交换。
```

**示例 3：需求分析阶段**

```
做 IFPUG 功能点估算，目标 40 个：
电商订单系统，含商品管理、订单流转、支付对接、物流跟踪、退换货处理。
```

### 触发判断（看是否生效）

Claude 调用 skill 时，会自动按这个流程：

1. **检查信息完整性**：若你没说功能点数量，会反问"目标拆分多少个？"
2. **拆模块**：先把系统按业务划分成模块
3. **找数据**：每个模块下先列 ILF/EIF
4. **延伸事务**：每个数据延伸出添加、修改、删除、查询、统计等 EI/EO/EQ
5. **生成两份文件**：md 和 xlsx 同时落盘

## 五、输出说明

### 表格格式

| 功能需求 | 序号 | 类型 type | 功能点数 number | 软件功能点名称 function_point_name |
|---------|------|-----------|----------------|----------------------------------|
| 客户管理 | 1    | ILF       | 7              | 客户信息                          |
| 客户管理 | 2    | EI        | 4              | 客户信息添加                      |
| 客户管理 | 3    | EQ        | 4              | 客户信息查询                      |
| ...     | ...  | ...       | ...            | ...                              |

**列含义**：

- **功能需求**：业务模块名（客户管理、订单管理等），用于按模块汇总
- **类型 type**：IFPUG 五种类型之一
- **功能点数 number**：该类型的标准功能点数值（详见下表）

**五类功能点的标准点数**：

| 缩写 | 含义 | 词性 | 单点功能点数 |
|------|------|------|-------------|
| ILF | 内部逻辑文件 | 名词 | 7 |
| EIF | 外部接口文件 | 名词 | 5 |
| EI  | 外部输入     | 动词 | 4 |
| EO  | 外部输出     | 动词 | 5 |
| EQ  | 外部查询     | 动词 | 4 |

### 输出文件

每次拆分自动在 **当前工作目录** 生成：

1. **`功能点清单.md`** — Markdown 表格，可被 `/bid-writer` 等下游 skill 引用
2. **`功能点清单.xlsx`** — Excel 文件，可直接用于工时估算、投标文档

两份文件**数据完全一致**，末尾都有汇总行：`合计：N 个功能点，共 M 功能点数`。

### 已存在同名文件时

skill 会问你"覆盖 / 追加 / 另存为"，不会直接覆盖你的旧文件。

## 八、常见问题

### Q1: `/plugin isn't available in this environment` 怎么办？

**原因**：你的 Claude Code 环境（多见于 IDE 扩展）没有 `/plugin` 命令。

**解决**：

- 关掉 IDE 扩展，在 PowerShell 里跑 `claude` 启动独立 CLI 即可使用 `/plugin`
- 或改用上面的 **方式 B：手动复制安装**

### Q2: 输出只有 md 没有 xlsx？

**原因**：你的环境没有 Python 或没装 openpyxl。

**解决**：

```powershell
pip install openpyxl
```

或者你也可以用 Excel 打开 `功能点清单.md`，复制表格后另存为 xlsx（手动）。

### Q3: GitHub 连不上 / push 失败 / marketplace add 报错？

**原因**：中国大陆访问 github.com 不稳定。

**解决**：开代理后再试。如果 Claude Code 自身也走不通，可以临时给 git 配代理：

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

（端口按你实际工具调整，Clash 默认 7890）

### Q4: 拆分结果里出现"用户调研""系统测试"这种？

不会。skill 已经设了边界：**只输出系统功能，不输出人为活动**。

判别原则：能写成"系统能做 X"的是功能点；只能写成"人去做 X"的不是。

如果偶发出现，可以回 Claude："这条是人为活动，请重新拆分"。

### Q5: 拆出来的数量比我要求的少？

直接说"再补充 10 个"或"还需要包含 XXX 方面的功能点"即可。Claude 会基于已有结果增补。

### Q6: 想把同一份输出按不同业务模块拆开成多个 sheet？

当前 xlsx 是单 sheet。如果有需要，告诉 Claude："按功能需求列拆成多个 sheet"即可即兴生成。

## 九、与 bid-writer 联动

如果你同时安装了 `/bid-writer` 斜杠命令（投标技术文件写作），fp-breakdown 产出的 `功能点清单.md` 可被自动引用：

- **需求分析章节** — 引用功能点总数和模块划分
- **功能实现章节** — 按模块逐项展开技术方案
- **工时估算** — 直接用合计功能点数 × 单价

典型工作流：

```
1. 准备 技术要求.md
2. fp-breakdown 拆分功能点 → 生成 功能点清单.md / .xlsx
3. /bid-writer 编写投标文件 → 自动引用功能点清单
```
